# Feature List

Copy this file to `features.md` and replace the examples with your app's actual features.
Each line is treated as one feature to test. Lines starting with `#` are ignored.

## Format
One feature per line. Be specific enough that an AI agent understands what to test.
The more context you give, the better the tests will be.

## Examples (replace these with your app's features)

- User registration with email and password
- Login with valid credentials redirects to dashboard
- Invalid login shows appropriate error message
- Password reset flow via email link
- User profile editing (name, avatar, bio)
- Creating a new post with title and body text
- Editing an existing post
- Deleting a post with confirmation dialog
- Comment on a post
- Search returns relevant results
- Pagination works on listing pages
- File upload accepts images and shows preview
- User settings page saves preferences
- Logout clears session and redirects to home
- 404 page displays for unknown routes
- Form validation shows errors for required fields
- Responsive layout works on mobile viewport
